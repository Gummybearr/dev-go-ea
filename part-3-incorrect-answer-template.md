# EA Part 3 오답노트 템플릿

기준일: 2026-04-24

이 템플릿은 Part 3 오답을 `authority / scope / deadline / form` 중심으로 남기기 위한 용도다.

## 사용 원칙

- 한 문제당 `3~6줄`
- 계산보다 `어느 절차 / 어느 권한 / 어느 폼`을 놓쳤는지만 적는다
- 아래 4개는 가능하면 매번 적는다:
  - stage
  - authority / scope issue
  - form / notice
  - trigger next time
- 오답은 아래 4개 중 하나로 태그한다:
  - `practices-procedures`
  - `representation-before-irs`
  - `specific-areas-representation`
  - `filing-process`

## 가장 짧은 기본 템플릿

```md
### [날짜] [문제번호 또는 출처]
```

## 자주 나오는 `Question type` 예시

- authority to practice
- conflict of interest
- sanctionable conduct
- due diligence
- representation authorization
- legal authority hierarchy
- collection remedy selection
- CDP / CAP stage
- audit / appeals stage
- amended return vs abatement
- e-file compliance
- rejected return timing

## `Why I missed it`를 쓰는 방식

나쁜 예:

- 헷갈림
- 실수
- 절차를 몰랐음

좋은 예:

- `8821`를 representation authorization으로 착각했다
- collection stage와 appeals stage를 같은 것으로 읽었다
- authoritative source와 참고용 source를 구분하지 않았다
- `1040-X`와 `843`의 역할을 섞었다
- rejected return 후 후속 기한을 놓쳤다

## `Trigger next time`를 쓰는 방식

좋은 예:

- `representation issue -> 2848 or 8821 first`
- `collection issue -> remedy + form + deadline`
- `audit notice -> RAR / CP2000 / 30-day / 90-day`
- `e-file issue -> ERO / EFIN / 8948 / 8879 / 8453`
- `ethics issue -> Circular 230 first`

## 시험 직전 오답 압축 규칙

- `low confidence`만 남긴다
- 같은 절차 혼동이 2번 이상 나오면 최종노트로 올린다
- 최종노트에는 `rule 문장 + trigger 문장`만 남긴다

예:

```md
- Representation issue: 2848 or 8821 first
- Collection issue: remedy + form + deadline
- Audit issue: identify notice stage first
- Appeals issue: 30-day and 90-day are not the same
- E-file issue: think 8948 / 8879 / 8453 / rejected-return timing
```
